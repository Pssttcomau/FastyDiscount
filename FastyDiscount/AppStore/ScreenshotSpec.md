# FastyDiscount — Screenshot Specifications

## Required Screenshot Sizes

App Store Connect requires screenshots for the following display sizes. Providing the
6.7" (iPhone) and 12.9" (iPad) sets is mandatory; Xcode scales them to cover smaller
sizes automatically unless you supply dedicated assets.

| Device Class | Display Size | Resolution (px) | Required |
|---|---|---|---|
| iPhone 15 Pro Max | 6.7" Super Retina XDR | 1290 × 2796 | Yes (mandatory) |
| iPhone 11 Pro Max | 6.5" Super Retina OLED | 1242 × 2688 | Yes (recommended) |
| iPhone SE (3rd gen) | 4.7" Retina | 750 × 1334 | Optional |
| iPad Pro (6th gen) | 12.9" Liquid Retina | 2048 × 2732 | Yes (mandatory for iPad) |
| iPad Pro (4th gen) | 11" Liquid Retina | 1668 × 2388 | Optional |
| Apple Watch Series 9 | 45mm | 396 × 484 | Yes (if Watch app submitted) |
| Apple Watch Ultra 2 | 49mm | 410 × 502 | Yes (if Watch app submitted) |

## Screenshot Production Notes

- Format: PNG (no alpha for iPhone/iPad; Apple rejects transparent PNGs)
- Colour space: sRGB
- No device frames required (App Store Connect adds them automatically for "framed" previews; submit raw screenshots)
- Add caption text using Figma / Sketch mockup template at the correct resolution
- Screenshots must NOT include pricing information that conflicts with the actual IAP price
- Screenshots must NOT show notification content not produced by the app itself
- For App Store previews (video), 15–30 second H.264 videos at the same resolution are optional

## App Icon Specification

| Usage | Size | File |
|---|---|---|
| App Store listing | 1024 × 1024 pt @1x PNG | AppIcon-1024.png |
| iPhone Home Screen | Auto-generated by Xcode from 1024 source | AppIcon.appiconset |
| iPad Home Screen | Auto-generated by Xcode | AppIcon.appiconset |
| Apple Watch | Separate AppIcon in Watch target | WatchAppIcon.appiconset |

### Icon Design Guidelines
- Flat design with FastyDiscount brand colours
- No rounded corners in source (iOS rounds automatically)
- No transparency / alpha channel
- Background fill is solid (no gradient that fades to transparent)
- Text should be minimal or absent at small sizes

## Launch Screen Configuration

Launch screen is configured via `UILaunchScreen` dictionary in Info.plist (no Storyboard).
Settings:
- `UIColorName`: `LaunchBackground` (defined in Assets.xcassets as brand primary colour)
- `UIImageName`: `AppIcon` (centred app icon shown on launch)
- No `UINavigationBar` or `UITabBar` (simple branded splash)

This approach is already configured via `INFOPLIST_KEY_UILaunchScreen_Generation = YES`
in the Xcode build settings, which tells Xcode to generate a default launch screen from
the app icon automatically.
